🎯 CORE CONCEPT

Example:
  User: "Find all VIP contacts and send them the renewal reminder"
    ↓
  Agent: [Parse] → [Plan] → [Validate] → [Execute]
    ↓
  Output: "✓ 14 sent, ✗ 1 failed"

✅ FUNCTIONAL REQUIREMENTS (6 areas)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
FR-1: Natural Language Understanding
  - Parse instructions into structured intent
  - Extract entities (contacts, tags, templates)
  - Handle multi-step instructions (2+ API domains)

FR-2: API Orchestration Planning
  - Generate execution plan from intent
  - Validate plan feasibility
  - Optimize plan (batch operations)
  - Support dry-run mode

FR-3: WATI API Integration (5 domains)
  - Contacts: search, add/update attributes, manage tags
  - Messages: send text, send template messages
  - Templates: list, get details
  - Broadcasts: create and send to segments
  - Operators: assign conversations, manage tickets

FR-4: Execution & Error Handling
  - Execute API calls in sequence
  - Handle partial failures gracefully
  - Provide detailed error messages
  - Support human-in-the-loop confirmation

FR-5: User Experience (CLI)
  - Rich formatting (colors, tables, progress bars)
  - Explain each step in plain language
  - Confirm destructive actions
  - Support interactive mode

FR-6: Mock/Real API Toggle
  - Realistic mock layer
  - Clearly swappable with real client

🔑 KEY DECISIONS & RATIONALE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. LangGraph vs. ReAct Agent
   ✓ Chose: LangGraph state machine
   Why: Deterministic plans, explicit control, human-in-the-loop support
   Trade-off: More boilerplate, but worth it for predictability

2. Claude 3.5 Sonnet vs. GPT-4
   ✓ Chose: Claude 3.5 Sonnet
   Why: Better tool calling, 200k context, comparable cost
   Trade-off: Vendor lock-in, but easy to swap via LangChain

3. Mock-First Development
   ✓ Chose: Build mock client first
   Why: Demo without credentials, faster iteration
   Trade-off: Mock may drift, mitigated by using real API docs

4. CLI vs. Web UI
   ✓ Chose: CLI with Rich
   Why: Time constraint (60+ min for web UI), showcases agent logic
   Trade-off: Less accessible, but acceptable for v1
